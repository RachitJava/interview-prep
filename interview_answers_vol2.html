🌳 Data Structures & Algorithms (Core Logic)

1. How do you solve the 'Two Sum' problem in O(N)?
Use a HashMap to store the value and its index as you iterate. Check if `target - current_val` exists in the map. If yes, you found the pair.
2. Difference between BFS and DFS? When to use which?
BFS (Queue): Explores level by level. Use for shortest path in unweighted graphs. DFS (Stack/Recursion): Goes deep then backtracks. Use for topological sort, cycle detection, or searching a deep tree.
3. What is a 'Sliding Window'? Give an example.
A technique to handle sub-arrays/sub-strings. Instead of nested loops O(N²), move a 'window' of two pointers. Example: Finding the longest substring without repeating characters.
4. Explain 'Binary Search' and its complexity.
Divides a sorted array in half each time. Complexity: O(log N). Used in every search-related interview problem.
5. What is a 'Trie' and why do we use it for autocomplete?
A tree structure where each node is a character. It's efficient for prefix searching. Finding all words starting with "app" takes O(Length of search string), independent of how many words are in the dictionary.

🏗️ System Design (Architectural Patterns)

6. How would you design a 'Rate Limiter'? (Token Bucket Algorithm)
Use a Token Bucket algorithm. A bucket holds N tokens, filled at a fixed rate. Each request consumes 1 token. If the bucket is empty, drop the request. Best implementation: Redis with `lua` scripts for atomicity.
7. SQL vs NoSQL: Which one for an E-commerce system?
SQL (Postgres/MySQL) for Transactions, Orders, and Payments (ACID compliance is critical). NoSQL (Mongo/Elastic) for Product catalogs (flexible schema) or Search results (fast reads).
8. What is 'Consistent Hashing'? Why is it used in Load Balancing?
A way to map keys to servers so that when a server is added or removed, only 1/N of keys move. Without it, adding a server causes a massive reshuffle, crashing the system. Critical for distributed caches like Redis clusters.
9. Describe the 'API Gateway' pattern.
A single entry point for all clients. It handles Authentication, Rate Limiting, and Routing to internal microservices. It hides the complexity of the internal network from the client.
10. How do you handle 'Read Heavy' systems at scale?
1. Caching (Redis/Memcached). 2. Database Replication (Read Replicas). 3. CDN for static assets. 4. Materialized Views in the DB.

☕ Java & Multithreading (Enterprise Depth)

11. Abstract Class vs Interface (Java 8+)?
Interface: For "Can-do" behavior (e.g., Runnable). Supports multiple inheritance. Can have default/static methods now. Abstract Class: For "Is-a" relationship. Can have state (fields) and constructor. Use for shared code between related classes.
12. Immutable Class in Java: How to make one?
1. Mark class as final. 2. All fields private and final. 3. No setter methods. 4. Use Deep Copy in constructors and getters for mutable fields (like Dates or Lists).
13. ConcurrentHashMap vs Hashtable?
Hashtable locks the ENTIRE map (slow). ConcurrentHashMap (Java 8) uses CAS (Compare-And-Swap) and synchronized only on the specific bucket being updated. Much faster for multi-core CPUs.
14. What is the 'Diamond Problem'? How does Java solve it?
Occurs when a class implements two interfaces having the same default method. Java forces the developer to override and specify which interface method to use using `InterfaceName.super.methodName()`.
15. Explain Java 8 Streams `map()` vs `flatMap()`.
map() transforms one element into another. flatMap() transforms one element into a stream and then flattens all streams into one. Use flatMap when you have a List of Lists.
16. What is a 'Deadlock'? How to find it in production?
Two threads waiting for each other's locks. To find: Use `jstack` or `jconsole` to take a Thread Dump. It will explicitly list "Found one Java-level deadlock".
