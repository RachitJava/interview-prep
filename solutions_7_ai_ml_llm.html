🤖 Solutions 7: AI/ML & LLM (30 Topics)

1. What is a Transformer? Attention mechanism? 🔥Top
Transformer is an architecture that replaced RNNs by using Self-Attention. It processes entire sequences at once (parallel) and calculates a score for how much every word should "pay attention" to every other word to understand context.
2. How does GPT work? Decoder-only? 🔥Top
GPT (Generative Pre-trained Transformer) is a **Decoder-only** model. It predicts the "next token" in a sequence based on all previous tokens. It is "unidirectional"—it only looks at tokens to its left.
3. What is tokenization? BPE vs WordPiece
Converting text into numeric IDs. BPE (Byte Pair Encoding) merges frequent character pairs. WordPiece (used by BERT) merges based on likelihood of training data. Both handle sub-words to solve the "out-of-vocabulary" problem.
4. Explain RAG pipeline 🔥Top
Retrieval-Augmented Generation. 1. Convert docs to embeddings. 2. Store in Vector DB. 3. At query time, find top-K most relevant docs. 4. Feed these docs + user query to LLM. Result: Fact-based generation without hallucinations.
5. What is LoRA? Reducing training cost 🔥Top
Low-Rank Adaptation. Instead of fine-tuning all 7B parameters, it adds small, low-rank matrices to the model layers. We update ONLY these small matrices. Reduces VRAM/Memory requirements by 10x-100x.
6. Knowledge Distillation 🔥Top
Training a small **Student** model (e.g. Llama 8B) to mimic the output probability distribution of a large **Teacher** model (e.g. GPT-4). Result: Small model with large model's logic.
7. What are embeddings? Vector DBs 🔥Top
Embeddings are high-dimensional vectors (arrays of numbers) reflecting "meaning". Vector DBs (Pinecone/Chroma) use algorithms like **HNSW** or **IVF** to perform extremely fast similarity searches on these vectors.
8. Prompt Engineering — chain-of-thought
Zero-shot: No examples. Few-shot: Give 3-5 examples. Chain-of-thought: Telling the AI "Let's think step by step" to improve reasoning and math accuracy.
9. Hallucination in LLMs 🔥Top
Generating plausibly sounding but false info. Reducers: RAG, lower Temperature, Citations/Sources, and Multi-turn verification.
10. LangChain — agents & tools
A framework to link LLMs with external data/tools. **Agents** can decide which tool to use (e.g. Google Search or Calculator) to finish a user's task autonomously.
11. Temperature vs Top_p
Temperature scales probabilities (0 is direct, 1 is random). Top_p (Nucleus sampling) selects the smallest set of tokens whose cumulative probability exceeds P.
12. Context window limits
Total tokens (Input + Output) a model can handle. For long docs, use **Chunking** or **Sliding Window** to feed only relevant parts to the model.
13. Fine-tuning vs RAG 🔥Top
Fine-tuning: Updates model weights for style/tone/specific tasks. RAG: Feeds fresh data to a frozen model. Use RAG for knowledge, fine-tuning for specialized behavior/formatting.
14. What is RLHF?
Reinforcement Learning from Human Feedback. Used to "align" models to be helpful/safe. Humans rank model outputs, and a Reward Model is trained to guide the LLM toward better answers.
15. Bias-variance tradeoff
Bias: Underfitting (model too simple). Variance: Overfitting (model too complex). Goal: Minimize both to generalize well to unseen data.
16. Precision vs Recall vs F1 🔥Top
Precision: How many predicted positives were correct? Recall: How many actual positives did you find? F1: Harmonic mean of both. In medical AI (SmartCPR), Recall is vital.
17. Overfitting — prevention
Happens when model learns noise. Prevention: Dropout, L1/L2 Regularization, Early Stopping, and adding more training data.
18. What is transfer learning?
Taking a pre-trained model (trained on huge data) and fine-tuning it for your specific, smaller dataset. Saves time and Millions in compute costs.
19. Supervised vs Reinforcement Learning
Supervised: Learns from (Input, Label) pairs. Reinforcement: Learns through trial and error, maximize a "Reward" signal in an environment.
20. Gradient Descent — Adam optimizer
Algorithm to minimize loss. **Adam** (Adaptive Moment Estimation) adjusts the learning rate for each parameter, making training faster and more stable.
21. CNN vs RNN vs Transformer
CNN (Images - spatial features), RNN (Text/Audio - sequential), Transformer (Text/Everything - parallelized context).
22. Confusion Matrix
A table visualizing model performance: True Positives (TP), True Negatives (TN), False Positives (FP/Type I error), and False Negatives (FN/Type II error).
23. Feature engineering
The process of creating better Input data (features) using domain knowledge. "Better data beats better models." (e.g. Heart Rate Variance instead of raw Pulse).
24. Model drift
Performance decay over time because real-world data changed (e.g. a fraud model failing because hackers changed tactics). Requires retraining.
25. Cross-validation (k-fold)
Splitting data into K parts. Training K times, each time using a different part as the test set. Ensures model quality isn't just "luck" based on a single split.
26. Batch normalization
Standardizing inputs to a layer for each mini-batch. Accelerates training by allowing higher learning rates and reducing dependency on initial weights.
27. Deploying an ML model 🔥Top
1. Serialize (Pickle/ONNX). 2. API (Flask/FastAPI). 3. Container (Docker). 4. Orchestrate (SageMaker/K8s). Use Batch production or Real-time inference.
28. A/B testing for ML
Directly comparing Model A vs Model B on real traffic. Measuring business metrics (e.g. Click-through rate) instead of just technical metrics (Accuracy).
29. FOLD-RM (SmartCPR logic) 🔥Top
An Explainable AI method. It creates **Human-readable Rules** from data. Unlike a "Black Box" Neural Network, FOLD-RM can explain *why* it decided a patient is in cardiac arrest.
30. AI Code Gen Pipeline (FBPM) 🔥Top
Step 1: Extract business rules. Step 2: Use GPT-4 to generate samples. Step 3: Distill to Llama using LoRA fine-tuning. Step 4: Verify generated code against rule engine.
