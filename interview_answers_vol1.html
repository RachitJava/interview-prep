🤖 Section 1: AI, LLM & GenAI (The Wow Factor)

1. Explain RAG (Retrieval-Augmented Generation). Why is it better than fine-tuning for most business cases?
"RAG provides an LLM with external, up-to-date knowledge without re-training."
  • How it works: We convert company docs into "embeddings" (vectors), store them in a Vector DB (Pinecone). When a user asks a question, we find relevant docs first and send them to the LLM as context.
  • Why it's better: 1. Zero training cost. 2. No "hallucinations" (the AI can cite its sources). 3. Real-time updates (just add a new doc to the DB).
2. What is LoRA (Low-Rank Adaptation) and how does it reduce training cost?
"LoRA is an efficient fine-tuning technique that trains only a tiny fraction of model weights."
  • Concept: Instead of updating all 7 billion parameters of a model, LoRA inserts small "adapter" layers. We freeze the original model and only train these tiny adapters.
  • Impact: Reduces VRAM requirements by 10x and model size by 10,000x. Can be done on a single consumer GPU.
3. Explain Knowledge Distillation in the context of your AmEx project.
"Training a small 'student' model to mimic a large 'teacher' model."
  • The Process: I used GPT-4 (Teacher) to generate high-quality API code samples from prompts. I then used this data to fine-tune Llama 3 (Student).
  • The Result: The resulting Student model has 90% of the Teacher's quality but costs 0$ in API fees because we host it ourselves.

🏗️ Section 2: System Design (High-Level Architecture)

4. Monolith vs Microservices: When would you choose which?
"Monolith for speed of initial development; Microservices for scale and team independence."
  • Monolith: Best for small teams or "0 to 1" startups. Simple to deploy and test.
  • Microservices: Best for large scale (like Sabre where I worked). Allows teams to deploy independently. If the "Billing Service" fails, the "Search Service" stays up.
5. How do you handle Distributed Transactions across microservices (Saga Pattern)?
"Using the Saga Pattern to maintain consistency without heavy locking."
  • Choreography: Each service performs its own transaction and publishes an event (e.g., OrderPlaced → PaymentService reacts).
  • Compensations: If Payment fails, the Payment service publishes "PaymentFailed" and the Order service "undoes" the order. No global locks needed.

☕ Section 3: Java & Spring Boot (Core Backend)

6. Explain the internal working of a HashMap. How are collisions handled?
"HashMap uses Hashing and Buckets (Array + LinkedList/Tree)."
  • Storage: It uses `hashCode()` to find an index in an internal array.
  • Collisions: If two keys have the same index, they are stored in a LinkedList (Chaining).
  • Java 8 Optimization: If a bucket grows beyond 8 elements, the LinkedList converts into a Red-Black Tree for O(log N) performance.
7. What is Spring Boot Auto-Configuration?
"Spring Boot attempts to automatically configure your app based on the 'jars' it finds on the classpath."
  • If `spring-boot-starter-data-jpa` is present, it automatically sets up a DataSource and EntityManager.
  • We can override it anytime using the `application.properties` file.

🗄️ Section 4: SQL & Database (Data Integrity)

8. Indexing: Why does it speed up reads but slow down writes?
"Indexes are like the Index page at the back of a book."
  • Reads: Instead of scanning the whole table (Full Table Scan), the DB looks at the B-Tree index to find the exact location in O(log N).
  • Writes: Every time you INSERT a row, the DB must also update the Index tree, which takes extra time and I/O.
9. Explain Transaction Isolation Levels.
"Controls how data changes made by one transaction are visible to others."
  • Read Uncommitted: Fast but risky (Dirty Reads).
  • Read Committed: Most common (Prevents Dirty Reads).
  • Repeatable Read: (MySQL Default) Prevents changes to data you've already read.
  • Serializable: Perfect consistency but very slow (highest locking).

🐍 Section 5: Python (AI & Scripting)

10. What is a Decorator in Python? Give a practical example.
"A function that wraps another function to add behavior without modifying its source code."
@timer_decorator def complex_calculation(): # do work...
  • Used for Logging, Authentication, or Timing. In my current project, I use them for measuring LLM response times.
11. Generators (yield) vs Lists: When to use what?
"Generators use 'Lazy Evaluation' to save memory."
  • List: Loads 1 million records into RAM immediately. Great if you need to access them multiple times.
  • Generator: Generates one record at a time as you loop. Perfect for processing huge LLM training datasets that don't fit in RAM.

☁️ Section 6: Cloud & DevOps (Deployment)

12. Why Docker? What's the difference between an Image and a Container?
"Docker solves the 'Works on my machine' problem."
  • Image: A snapshot/blueprint of your app (OS + Runtime + Code). It's read-only.
  • Container: A running instance of that Image.
13. Kafka: What is a Consumer Group?
"A way to scale message consumption by distributing partitions among multiple consumers."
  • If you have 10 partitions and 5 consumers in a group, each consumer handles 2 partitions. This allows parallel processing of massive billing streams at Verizon.

🌳 Section 7: DSA (The Logic Core)

14. How do you find a cycle in a Linked List? (Floyd’s Cycle Finding)
"Using two pointers: Slow and Fast."
  • The Slow pointer moves 1 step; Fast pointer moves 2 steps.
  • If they ever meet, there is a loop. If Fast reaches `null`, there is no loop.
  • Time: O(N) | Space: O(1)
15. What is Dynamic Programming (DP) and when to use it?
"Breaking a big problem into subproblems and storing results (Memoization)."
  • Use it when you see **Overlapping Subproblems** (calculating the same thing twice) and **Optimal Substructure**.
  • Common example: Fibonacci or Knapsack problem.
