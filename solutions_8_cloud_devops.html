☁️ Solutions 8: Cloud, DevOps & Kafka (30 Topics)

1. Docker — images vs containers 🔥Top
Image: Read-only snapshot of your app (blueprint). Container: A running instance of that image. You can run multiple containers from one image.
2. Docker Compose
A tool for defining and running multi-container applications (e.g. App + DB + Redis). Uses a `docker-compose.yml` to orchestrate how they network together.
3. Kubernetes — pods, services, deployments 🔥Top
Pod: Smallest unit (holds container). Deployment: Manages Pods (scaling/rolling updates). Service: Exposes Pods to network (load balancer). Namespace: Logical isolation.
4. K8s — auto-scaling (HPA)
Horizontal Pod Autoscaler. Automatically increases or decreases the number of pods based on observed CPU utilization or custom metrics from Prometheus.
5. K8s — rolling update vs canary
Rolling: Slowly replaces old pods with new ones. Canary: New version is released to 5% of users; if stable, roll out to 100%. Blue-Green: Two identical environments, switch traffic instantly.
6. CI/CD pipeline design 🔥Top
1. Build (Docker build) 2. Unit Test 3. Security Scan (Snyk/Sonar) 4. Staging Deploy 5. Approval 6. Production Deploy. Done via GitHub Actions or Jenkins.
7. Jenkins — pipeline-as-code
Defining build steps in a **Jenkinsfile** (Groovy syntax) stored in Git. Ensures the pipeline is version-controlled, just like the application code.
8. Terraform — state management
Terraform uses a **State File** to track what resources exist in AWS. Always use Remote State (S3 + DynamoDB locking) so multiple developers don't conflict.
9. AWS EC2 auto-scaling groups
Automatically launches or terminates EC2 instances to maintain a specific "Desired Capacity" or respond to traffic spikes (e.g. High CPU).
10. AWS S3 storage classes
Standard (high speed), IA (infrequent access), Glacier (long-term archive). Choosing the right class can cut AWS costs by 80%.
11. AWS Lambda — cold starts 🔥Top
Delay when a function is called after a gap. AWS must provision a container. **Reduction**: Use 'Provisioned Concurrency', keep package size small, or use lightweight runtimes (Node/Go).
12. AWS RDS — read replicas
Creates copies of your DB for **Read-only** traffic. Offloads the main Master DB so it can focus on Writes. Good for scale.
13. AWS SageMaker
AWS's platform for building, training, and deploying ML models. Handles the scaling of GPU training clusters and real-time inference endpoints.
14. VPC & Subnets basics
VPC: Your private network in AWS. Subnets: Dividing VPC into ranges. **Public Subnet**: Connected to internet gateway. **Private Subnet**: No direct internet access (safe for DBs).
15. IAM — least privilege
Giving users/roles ONLY the exact permissions they need to do their job. Avoid `AdministratorAccess`; use specific policies like `S3ReadOnlyAccess`.
16. Kafka — topics & partitions 🔥Top
Topic: Name of the stream. Partition: Division of topic for parallelism. Each partition can only be read by ONE consumer in a group at a time.
17. Kafka — message ordering
Ordering is only guaranteed **within a partition**. To ensure ordering for a specific user, use `UserID` as the **Partition Key** so all their events land in the same partition.
18. Kafka vs RabbitMQ 🔥Top
Kafka: Distributed log, messages are kept after being read, massive throughput. RabbitMQ: Traditional queue, messages deleted after read, complex routing/priority.
19. Kafka — idempotent producer
Ensures a message is written exactly once even if network retries happen. Prevents duplicate data (e.g. double-charging a customer).
20. What is a service mesh (Istio)?
A layer between microservices that handles **Traffic splitting, Observability, and Retries** automatically. Uses "Sidecar Containers" to intercept all network calls.
21. 12-Factor App 🔥Top
Best practices for cloud apps. Key ones: **Codebase** (version control), **Config** (env vars), **Backing Services** (DB/Queue as attached resources), **Statelessness**.
22. Observability — logs, metrics, traces
Logs (what happened), Metrics (performance stats), Traces (how a single request traveled through 10 microservices).
23. Infrastructure as Code (IaC)
Using tools like **Terraform** or **CloudFormation** to write code that builds your servers. Benefits: Audit trail, version control, and replicable environments.
24. How to handle secrets?
Never store in Git. Use **HashiCorp Vault**, **AWS Secrets Manager**, or K8s Secrets. Inject into apps at runtime via Environment Variables.
25. What is GitOps?
A DevOps practice where the "Source of Truth" for infrastructure is a Git repository. Tools like **ArgoCD** watch Git and ensure the K8s cluster matches the code.
26. Sync vs Async communication
Sync (REST): Immediate response. Async (Kafka): Send and forget. Use Async to decouple services and handle spikes (e.g. Invoice generation taking 5 mins).
27. API versioning strategies
**Header Versioning** (`Accept: v2`), **URL Versioning** (`/api/v1/user`), or **Query Params**. URL versioning is the most common for public APIs.
28. Liveness vs Readiness probes
Liveness: Is the container dead? (Restart if failed). Readiness: Is the app ready to handle traffic? (Don't send users if it's still booting up).
29. How to debug a production issue? 🔥Top
1. Isolate (Is it one user or all?). 2. Observe (Check Grafana for CPU/Memory spikes). 3. Drill down (Check ELK logs). 4. Fix & Record (Post-mortem to avoid recurrence).
