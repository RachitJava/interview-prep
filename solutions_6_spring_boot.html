🍃 Solutions 6: Spring Boot (30 Topics)

1. What is Spring IoC container? Bean lifecycle? 🔥Top
IoC Container handles object creation, wiring, and lifecycle. Lifecycle: 1. Instantiate 2. Populate Props 3. `BeanNameAware` 4. `BeanFactoryAware` 5. Pre-Initialization (PostProcessor) 6. `InitializingBean.afterPropertiesSet` 7. Custom Init 8. Post-Initialization.
2. @Autowired — types of DI 🔥Top
Constructor (Recommended: best for testability and immutable fields), Setter (for optional deps), Field (uses reflection, discouraged).
3. @Component vs @Service vs @Repository vs @Controller 🔥Top
@Component: General purpose bean. @Service: Business logic. @Repository: Persistence (auto-catches DB exceptions). @Controller: Web MVC. All are `@Component` specializations.
4. Bean scopes — singleton to session
Singleton (One per container - default), Prototype (One per request to getBean), Request (One per HTTP request), Session (One per HTTP session).
5. What is @Transactional? Implementation details 🔥Top
Uses Spring AOP Proxy to wrap methods. If method completes, commit; if exception (Runtime), rollback. Propagation types: `REQUIRED` (Join current), `REQUIRES_NEW` (Suspend current, start new).
6. Spring Boot auto-configuration — how does it work?
Scans `META-INF/spring.factories` or `org.springframework.boot.autoconfigure.AutoConfiguration.imports`. It uses `@ConditionalOnClass` or `@ConditionalOnMissingBean` to decide what to enable.
7. @RestController vs @Controller
@RestController = `@Controller` + `@ResponseBody`. Automatically converts returned objects to JSON/XML instead of looking for an HTML template.
8. Exception handling — @ControllerAdvice 🔥Top
A global interceptor for exceptions. Allows you to define `@ExceptionHandler` in one place rather than in every controller, returning clean, consistent error JSONs.
9. Spring Security — JWT authentication flow 🔥Top
1. Client sends login. 2. Filter authenticates and returns **JWT**. 3. Client sends JWT in `Authorization` header for next calls. 4. Filter validates token signature without hitting DB.
10. Spring Data JPA — Specifications
Allows creating reusable, dynamic 'Where' clauses for queries using the Criteria API. Great for multi-filter search screens.
11. What is Spring AOP? Use cases
Aspect-Oriented Programming. Separates cross-cutting concerns from business logic. Uses: Logging, Security, Transaction Management.
12. Profiles — application-dev.yml
Groups configuration settings for different environments. Activate using `-Dspring.profiles.active=prod` to switch DB URLs or API keys easily.
13. Spring Boot Actuator
Provides production-ready features: `/health`, `/info`, `/metrics`, `/prometheus`. Essential for monitoring with Grafana.
14. @Async — implementation
Executes a method in a separate thread. Requires `@EnableAsync` and returns a `Future` or `CompletableFuture`. Ensure calling from a different bean for Proxy to work.
15. Spring WebFlux vs MVC
MVC: Servlet-based, blocking (Thread-per-request). WebFlux: Reactive, non-blocking (Event-loop, Netty). Use WebFlux for high-concurrency streaming apps.
16. How to handle N+1 query problem? 🔥Top
Use Join Fetch in JPQL queries, `@EntityGraph`, or configuring Batch Fetching in properties to load associations in one go.
17. @Cacheable — Redis
Stores the result of a method in a cache. If method called with same params, returns cached value. Use `@CacheEvict` to clear cache when data updates.
18. Circuit breaker — Resilience4j
Limits impact of downstream service failure. Stages: **Closed** (allowing calls), **Open** (blocking calls, fail-fast), **Half-Open** (testing if service is back).
19. Spring Cloud — Service Discovery
Eureka allows microservices to find each other by name (e.g. `inventory-service`) instead of IP:Port, which can change in K8s.
20. Deploying Spring Boot in K8s
Package as **Docker image**, define Deployment with Liveness/Readiness probes, and use a Service (ClusterIP/LoadBalancer) for traffic.
21. @Scheduled — cron jobs
Runs tasks at fixed intervals. Supports `fixedDelay`, `fixedRate`, and `cron` expressions (e.g. every midnight).
22. Filter vs Interceptor vs AOP
Filter: Low level (Servlet container). Interceptor: Spring level (Handler mapping). AOP: Method level (Business logic).
23. Spring Batch — large data
Framework for high-volume batch processing. Follows **ItemReader → ItemProcessor → ItemWriter** pattern with automatic restart/skip features.
24. Testing in Spring Boot 🔥Top
@SpringBootTest: Full context. @DataJpaTest: Slice for DB testing. @MockBean: Mocking dependent services. @WebMvcTest: Slice for Controller testing.
25. OpenAPI/Swagger
Auto-generates interactive API documentation. Viewable at `/swagger-ui/index.html`. Essential for frontend developers to test headers.
26. Connecting to Kafka
Use `spring-kafka`. Inject `KafkaTemplate` to send messages and use `@KafkaListener` to consume them with automatic JSON deserialization.
27. Connection pooling — HikariCP
Default pool in SB. Highly tuned for performance. Important settings: `maximum-pool-size`, `connection-timeout`, `idle-timeout`.
28. REST vs gRPC vs Messaging
REST (HTTP/1, JSON): Standard, good for public APIs. gRPC (HTTP/2, Protobuf): Very fast, good for internal. Messaging (Kafka): Asynchronous, eventual consistency.
29. Spring Boot 3 & Jakarta EE 🔥Top
SB 3 requires Java 17+. Package names moved from `javax.*` to `jakarta.*`. Adds native image support via GraalVM for ultra-fast startup.
30. Distributed transactions — Saga Pattern
Microservices shouldn't share transactions. Use **Saga** (series of local TX) or **Transactional Outbox** pattern to ensure consistency across separate databases.
