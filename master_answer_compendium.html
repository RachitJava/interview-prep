🌳 DSA Selection (Questions 1-50) Detailed Solutions ↗

1
Find Two Sum in Array?
Use HashMap to store `(target - num)`. If difference exists in map, return indices. O(n) time.
2
Reverse a Linked List?
Use 3 pointers (prev, curr, next). Iteratively set `curr.next = prev`. O(n) time.
3
Detect Cycle?
Floyd’s Cycle Finding (Tortoise & Hare). Fast moves 2 steps, slow moves 1. Meeting = Cycle.
4
LRU Cache?
HashMap + Doubly Linked List. Map for O(1) access; DLL for tracking usage order.
5
Dijkstra's Algorithm?
Shortest path from source to all nodes using a PriorityQueue + Adjacency List. O(E log V).

6
Design a URL Shortener?
API Gateway + Base62 encoding + NoSQL DB. Use a Snowflake ID Generator for unique keys.
7
Scaling a Newsfeed (Twitter)?
Fan-out: push celebrity tweets to read-cache (pull model) vs push regular users (push model).
8
CAP Theorem?
In distributed systems, you only get 2 of 3: Consistency, Availability, Partition Tolerance.
9
Consistent Hashing?
Mapping keys to a ring. When nodes change, only 1/N keys move. Prevents cache thrashes.
10
Circuit Breaker Pattern?
Prevents cascading failures. If service A is down, B stops calling it immediately to save resources.

☕ Java Selection (Questions 101-150) Detailed Solutions ↗

1
Method Overloading vs Overriding?
Overloading: Same name, different params (Compile-time). Overriding: Same name & params in subclass (Runtime).
2
Diamond Problem?
When a class inherits from two parents with same method. Java solves this via Interfaces and explicit overriding.
3
Immutability?
Object state can't change after creation. String is immutable for Security, Caching, and Thread-Safety.
4
HashMap Working?
Uses hashing to find bucket. Collisions handled by LinkedList (before Java 8) and Balanced Tree (after Java 8).
5
Fail-fast vs Fail-safe?
Fail-fast: Throws ConcurrentModificationException immediately. Fail-safe: Works on a copy of the collection (e.g. CopyOnWriteArrayList).
6
Thread vs Runnable?
Thread is a class; Runnable is an interface. Use Runnable to support multiple inheritance.
7
Volatile keyword?
Guarantees that a variable is read from Main Memory, not CPU cache. Ensures visibility across threads.
8
JIT Compiler?
Just-In-Time. Compiles bytecode to native machine code at runtime for repeated methods to boost speed.
9
G1 Garbage Collector?
Region-based collector. Balances throughput and latency; suitable for large heaps with low pause times.
10
Records in Java 17?
Compact class declaration for Data Transfer Objects (DTOs). Auto-generates getters, equals, hashCode, and toString.

🐍 Python Selection (Questions 151-200) Detailed Solutions ↗

11
*args vs **kwargs?
*args: Variable positional arguments (Tuple). **kwargs: Variable keyword arguments (Dictionary).
12
Global Interpreter Lock (GIL)?
A lock that allows only one thread to control the Python interpreter. Limits CPU-bound multithreading.
13
Decorators?
Function that takes another function and extends its behavior. Useful for Logging/Auth.
14
Generators?
Functions that use yield. They return an iterator that generates values on the fly to save memory.
15
Mutable vs Immutable?
Mutable: List, Dict, Set. Immutable: Int, Float, String, Tuple.
16
@staticmethod vs @classmethod?
@classmethod knows about the class (receives 'cls'); @staticmethod does not (just a utility function).
17
Context Managers (with)?
Ensures resources (like files) are automatically closed after use. Uses __enter__ and __exit__.
18
Pickle vs JSON?
JSON: Human-readable, language-independent. Pickle: Binary, Python-specific, but insecure for untrusted data.
19
Monkey Patching?
Changing or extending code at runtime without modifying the source file. Use with caution.
20
List Comprehension?
Shorthand for creating lists. `[x*x for x in range(10)]`. Faster and more readable.

🗄️ SQL & Databases (Questions 201-250) Detailed Solutions ↗

21
WHERE vs HAVING?
WHERE filters before aggregation; HAVING filters after aggregation (used with GROUP BY).
22
B-Tree vs Hash Index?
B-Tree: Good for range queries (=, <, >). Hash: Extremely fast for exact equality (=) only.
23
Primary vs Unique Key?
Primary: Unique, Not Null, one per table. Unique: Unique, can have one NULL, multiple per table.
24
Normalization (1NF/2NF/3NF)?
1NF: Atomic values. 2NF: No partial dependency. 3NF: No transitive dependency. Reduces redundancy.
25
Database Sharding?
Horizontal partitioning—splitting large table across multiple physical databases to handle high traffic.
26
ACID vs BASE?
ACID (SQL): Strict consistency. BASE (NoSQL): Basically Available, Soft state, Eventual consistency.
27
Materialized View?
A view that stores the result of a query physically. Needs refreshing. Great for complex, slow queries.
28
Deadlock in DB?
When two transactions are stuck waiting for each other's locks. DB engine detects and kills one of them.
29
PostgreSQL vs MySQL?
Postgres: Highly extensible, ACID focus, handles complex queries better. MySQL: Extremely fast for simple reads/writes.
30
Composite Index?
An index on multiple columns. Order matters! `(col1, col2)` works for col1 alone, but not col2 alone.

🍃 Spring Boot (Questions 251-280) Detailed Solutions ↗

31
Bean Lifecycle?
Instantiate → Populate Props → Custom Init → **Usage** → Custom Destroy.
32
Qualifer vs Primary?
@Primary is the default choice. @Qualifier selects a specific bean by name.
33
Spring Security Filters?
Chain of interceptors handles auth. `UsernamePasswordAuthenticationFilter` is the key for login.
34
Actuator?
Monitoring tool. Provides endpoints like `/health`, `/metrics`, and `/env` for production health stats.
35
HikariCP?
The default Connection Pool in Spring Boot. It's the fastest and most efficient pool for DB connections.

🤖 AI & Machine Learning (Questions 281-310) Detailed Solutions ↗

36
Transformer Attention?
Scores how much "weight" to give each word in a sequence relative to the target word. Self-attention is the core.
37
Temperature in LLMs?
Controls randomness. 0.0 is deterministic (fixed); 1.0 is creative/random. Use 0.0 for structured code gen.
38
Hallucination?
AI generates facts that aren't true. Use RAG (Retrieval) to ground the output in actual company documents.
39
Fine-Tuning vs RAG?
Fine-Tuning: Changes model behavior/tone. RAG: Adds new knowledge without training. RAG is better for live data.
40
Vector Database?
Stores data as numbers (embeddings). Allows **Semantic Search**—find "car" when the user searches "vehicle".

☁️ Cloud & DevOps (Questions 311-340) Detailed Solutions ↗

41
Docker Image vs Container?
Image: Read-only blueprint of an app. Container: A running instance of that image.
42
K8s Pod?
Smallest unit. Houses one or more containers. Shares the same network and storage volume.
43
Helm?
The **Package Manager** for Kubernetes. Uses Charts to define, install, and upgrade complex K8s apps.
44
Canary Deployment?
Rolling out a new version to only 5% of users first. If no errors, roll out to 100%. Safety first.
45
Zero-Downtime Deployment?
Using **Blue-Green** or **Rolling Updates** to ensure users are never disconnected while code switches.
