🎯 Interview Master Answer Guide — Volume 3

Focus: SQL, Python, Cloud & infrastructure

🗄️ SQL & Databases (Tuning & Logic)

1. WHERE vs HAVING? WHERE filters rows BEFORE grouping. HAVING filters groups AFTER the GROUP BY operation.
2. INNER vs LEFT vs RIGHT Join? INNER: Only matching rows. LEFT: All from left + matching from right. RIGHT: All from right + matching from left.
3. What is a Window Function? Performs calculation across set of rows related to current row. Example: `RANK() OVER(ORDER BY salary DESC)` to find Nth highest salary.
4. How to find 2nd highest salary? `SELECT salary FROM emp ORDER BY salary DESC LIMIT 1 OFFSET 1;` or use a subquery with `MAX()`.
5. Database Sharding? Horizontal partitioning. Split one big table into multiple DBs based on a key (e.g., `user_id % 10`). Scalability.
6. What is ACID? Atomicity (all or nothing), Consistency, Isolation, Durability. Ensures reliable transactions.

🐍 Python for AI & Backend

7. What is the GIL? Global Interpreter Lock. Prevents multiple native threads from executing Python bytecodes at once. Limits CPU-bound multithreading. Solution: Multiprocessing.
8. *args vs **kwargs? *args: Pass variable number of non-keyword arguments (as tuple). **kwargs: Pass variable number of keyword arguments (as dict).
9. List vs Tuple? List: Mutable, slower, more memory. Tuple: Immutable, faster, less memory (fixed size).
10. Asyncio in Python? Uses a single-threaded event loop to handle concurrent I/O (network calls). `async` / `await` syntax. Great for high-concurrency APIs like FastAPI.
11. Deep copy vs Shallow copy? Shallow: Copies object reference (nested objects shared). Deep: Recursively copies everything (totally independent).
12. Python Decorators? Wrappers to modify function behavior. Used for @logging, @auth, @timing.

☁️ Cloud, DevOps & Kafka

13. Kafka Partitions? Basis for parallelism. One partition can only be read by ONE consumer in a group. More partitions = More throughput.
14. CI/CD Pipeline Stages? 1. Build (Compile code) → 2. Test (Unit/Integration) → 3. Security Scan → 4. Deploy (to Staging/Prod).
15. Terraform 'State' file? A JSON file tracking current infra. Tells Terraform what exists so it only creates/updates what's NEW. Never delete it!
16. Docker Image vs Layer? Images are made of Layers. Each Dockerfile command (RUN, COPY) creates a layer. Layers are cached to speed up builds.
17. Kubernetes 'Pod'? Smallest deployable unit in K8s. Usually contains ONE container. Pods are ephemeral (they die and restart).
18. AWS Lambda 'Cold Start'? Delay when a function is triggered after inactivity. AWS must spin up a new container. Solve with 'Provisioned Concurrency'.

🍃 Spring Boot (The Engine)

19. @Autowired vs Constructor DI? Constructor DI (Recommended): Better for testing, allows 'final' fields, prevents circular dependency.
20. What is @Transactional? Wraps a method in a DB transaction. If any exception occurs, it rolls back all changes automatically.
21. Spring Boot 'Starters'? Curated sets of dependencies. `spring-boot-starter-web` gives you Tomcat, Spring MVC, and JSON support in one click.
22. N+1 Query Problem? When Hibernate runs 1 query for parent + N queries for children. Fix: Use `JOIN FETCH` in JPQL or `@EntityGraph`.
