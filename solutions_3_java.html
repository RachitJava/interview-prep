☕ Solutions 3: Java (50 Topics)

Core Java & OOP

1. What are the 4 OOP pillars? 🔥Top
Abstraction (hiding complexity), Encapsulation (data hiding), Inheritance (code reuse), and Polymorphism (one interface, multiple forms). Example: A "Bank" interface with different "CreditCard" implementations.
2. Abstract class vs Interface — when to use which? 🔥Top
Use Interface for "behavioral" contracts (e.g., Runnable). Supports multiple inheritance. Use Abstract Class for shared "state" and default implementation between related classes.
3. What is method overloading vs overriding?
Overloading: Same method name, different parameters in the same class (Compile-time). Overriding: Subclass provides specific implementation for a method defined in parent (Runtime).
4. What is polymorphism? Compile-time vs runtime?
Ability of an object to take on many forms. Compile-time: Overloading. Runtime: Method Overriding (decided by the object type at execution).
5. Explain SOLID principles 🔥Top
Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion. They ensure code is maintainable and extensible.
6. What is the diamond problem? How does Java solve it?
When a class implements two interfaces with the same default method. Java forces the developer to explicitly override that method to resolve ambiguity.
7. What are access modifiers? Default vs protected?
Private (class only), Default (package only), Protected (package + subclasses), Public (everywhere).
8. Explain static keyword — variable, method, block, class
Belongs to the class rather than any specific instance. Static variables/methods can be accessed without creating an object. Static blocks run once when class is loaded.
9. What is the difference between == and .equals()?
== compares the memory address (reference). .equals() compares the content/value of the objects (needs to be overridden for custom objects).
10. What is immutability? Why is String immutable? 🔥Top
Object state cannot change after creation. String is immutable for Security (SQL injection), Caching (String Pool), and Thread-safety.

Collections Framework

11. ArrayList vs LinkedList — when to use? 🔥Top
ArrayList: Better for search/read (O(1)). LinkedList: Better for frequent insertions/deletions at start/end (O(1) if you have the node).
12. HashMap internal working 🔥Top
Uses hashing to find an index (bucket). If multiple keys map to same bucket (collision), they are stored in a LinkedList. In Java 8, if a bucket reaches 8 elements, it converts to a Balanced Tree (Red-Black Tree).
13. HashMap vs ConcurrentHashMap 🔥Top
HashMap is not thread-safe. ConcurrentHashMap allows concurrent threads to read/write using bucket-level locking (Java 7) or CAS (Java 8+). Much more performant than SynchronizedMap.
14. HashSet vs TreeSet vs LinkedHashSet
HashSet: Random order (fastest). TreeSet: Sorted order (uses TreeMap). LinkedHashSet: Insertion order maintained (uses Doubly-linked list).
15. How does HashMap handle collisions?
Via Chaining (using a LinkedList/Tree). In Java 8, it also performs Treeification when a bucket grows too large to keep search time O(log N).
16. Contract between hashCode() and equals()?
If two objects are equal according to `equals()`, they **must** have the same `hashCode()`. However, two objects with the same `hashCode()` are not necessarily equal.
17. Comparable vs Comparator
Comparable: Natural sorting (modify class, implement `compareTo`). Comparator: Custom sorting (external class, implement `compare`).
18. Fail-fast vs fail-safe iterators
Fail-fast: Throws `ConcurrentModificationException` if collection is modified during iteration. Fail-safe: Works on a copy (e.g. `CopyOnWriteArrayList`), so no exception.

Multithreading & Concurrency

19. Thread vs Runnable vs Callable 🔥Top
Thread: Class to extend. Runnable: Interface with `run()` (no return). Callable: Interface with `call()` that can return a result and throw checked exceptions.
20. synchronized keyword — method vs block level
Method: Locks the entire object. Block: Locks only a specific part of code. Block-level is more efficient as it reduces wait time for other threads.
21. What is a deadlock? How to prevent it? 🔥Top
Situation where two threads are waiting for each other's locks forever. Prevention: Ensure all threads acquire locks in the same fixed order.
22. volatile keyword — what does it guarantee?
Guarantees Visibility. Ensures that variable reads/writes go to main memory, not CPU cache. Doesn't guarantee atomicity.
23. Executor framework — ThreadPool
A managed pool of threads. Instead of creating expensive new threads, reuse idle ones. Types: Fixed, Cached, Scheduled.
24. CompletableFuture
Used for Asynchronous programming. Allows chaining of tasks without blocking the main thread. Supports functional methods like `thenApply`, `thenCombine`.
25. What is the happens-before relationship?
A memory consistency property. Guarantees that memory writes by one specific statement are visible to another specific statement across threads.
26. ReentrantLock vs synchronized
ReentrantLock offers more features: Fairness, `tryLock()` (non-blocking), and multiple condition variables.
27. Producer-Consumer problem
Solved using a BlockingQueue. When queue is full, producer waits; when empty, consumer waits. Handles all synchronization internally.

Java 8+ Features

28. Lambda expressions 🔥Top
Concise way to implement single-method interfaces (Functional Interfaces). Enables functional programming in Java.
29. Stream API 🔥Top
Allows pipeline processing of collections using map, filter, reduce. Doesn't change source data.
30. Optional 🔥Top
A container object used to represent either a value or "empty". Forces the developer to handle the `null` case explicitly.
31. Functional interfaces
Interfaces with exactly one abstract method. Predicate (test), Function (transform), Consumer (action), Supplier (provide).
32. Method references — :: operator
Syntactic sugar for lambdas that call an existing method. Example: `System.out::println` instead of `x -> System.out.println(x)`.
33. Default methods in interfaces
Allows interfaces to have method bodies. Added in Java 8 to evolve existing interfaces without breaking implementing classes.
34. Stream parallel() — pitfalls
Divides steam into sub-streams for multi-core processing. Pitfalls: Overhead for small lists, non-thread-safe state modification, order matters.

JVM & Memory

35. JVM architecture 🔥Top
ClassLoader (loads classes), Runtime Data Areas (Heap, Stack, PC), Execution Engine (JIT, GC).
36. Heap vs Stack memory
Heap: Stores objects, global access, GC handles it. Stack: Stores local variables and method calls, thread-specific, LIFO order.
37. Garbage Collection types
Serial (single-thread), Parallel (multi-thread throughput), G1 (low latency, balanced), ZGC (ultra-low latency for large heaps).
38. What is memory leak in Java?
When objects are no longer used but still referenced. Common causes: Static fields, unclosed listeners, long-running threads.
39. String pool — how does it work?
A special memory area in the Heap that stores unique string literals. Prevents creating duplicate strings to save memory.
40. ClassNotFoundException vs NoClassDefFoundError
Exception: Happens at runtime when trying to load via class name. Error: Happens when the class was present at compile-time but missing at runtime.

Design Patterns & Best Practices

41. Singleton pattern 🔥Top
Ensures a class has only one instance. Thread-safe implementation: Double-checked locking or using an Enum (most secure).
42. Factory vs Abstract Factory
Factory: Creates one type of object. Abstract Factory: Creates "families" of related objects without specifying their concrete classes.
43. Builder pattern
Used for creating complex objects with many optional parameters. Prevents "telescoping constructors" and makes code readable.
44. Observer pattern
One-to-many dependency. When one object changes state, all its dependents are notified. Example: Publisher-Subscriber models.
45. Strategy pattern
Defines a family of algorithms and makes them interchangeable at runtime. Example: Choosing different payment methods (PayPal/Card).
46. Decorator pattern
Wraps an object to add new functionality without changing its structure. Example: `BufferedInputStream` decorating `FileInputStream`.
47. Exception handling best practices 🔥Top
Don't catch `Exception`. Be specific. Use custom exceptions for business logic. Always close resources in `finally` or use try-with-resources.
48. What is try-with-resources?
Closing resources automatically if they implement `AutoCloseable`. Avoids manual `finally` blocks and memory leaks.
49. Serialization
Saving an object's state to byte stream. `transient` keyword prevents a field from being serialized (useful for passwords).
50. What's new in Java 17? 🔥Top
Records (immutable data classes), Sealed Classes (restricted hierarchy), Pattern Matching for Switch, and removal of old features.
